## 需求目标
- 添加多模型翻译

## 需求细节
- 设置页面新增 "模型" section
- 模型 section 以列表的形式呈现多个模型厂商（如 DeepSeek、OpenAI、Gemini、豆包等）
    - 每个模型厂商 Ïitem 从左至右展示：展开收起图标、模型厂商 icon、模型厂商名称、应用状态（如果当前模型被选择则在厂商名称后用展示文本“已选择”）
    - 在鼠标浮动到模型厂商 item 或者展开当前 item 时，item 最右侧展示按钮：应用
        - 应用：样式为与主题保持一致的主按钮，点击后应用当前厂商已选择模型进行翻译工作（只能单选），如未填写 API key 默认 disable 并在按钮左侧提示: "未填写 API Key"
    - 每个模型厂商 item 支持展开和收起
        - 展开后展示两个可输入 field：“模型选择”、“API Key” 以及一个按钮：验证，各占一行
        - 模型选择：代表当前厂商可支持哪些模型供选择，可支持搜索，默认为第一个模型
        - API Key： 和当前已有 deepseek api key 的控件一致
        - 验证：样式为与主题保持一致的副按钮，点击后验证当前模型厂商是否可连接成功，验证过程中按钮展示 loading 动画，验证后将结果展示在按钮左侧，验证结果无需保存，在关闭设置界面后重制
- 为多个模型厂商添加调用方法，在设置页面切换模型后应用相应模型实施翻译功能
    
## 支持的模型厂商及模型
- Deepseek
    - deepseek-chat
- 豆包
    - doubao-seed-translation
- Gemini
